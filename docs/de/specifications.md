# Spezifikationen

Die Engineering-Specs für dieses Paket liegen im Repository unter [`spec/`](https://github.com/nolte/vale-style/tree/main/spec) — nicht in dieser Docs-Seite. Jede Spec hat eine kanonische englische Fassung und eine deutsche Übersetzung, die strukturell synchron bleiben müssen (gleiche Überschriften, gleiche Anforderungs-Bullets, gleiche Akzeptanzkriterien).

## Wie man eine Spec liest

Specs folgen einer festen Vorlage: **Context**, **Goals**, **Non-Goals**, **Requirements** (RFC-2119-Schlüsselwörter — `MUST`, `SHOULD`, `MAY`), **Acceptance Criteria**, **Open Questions**. Eine Anforderung in einer Spec ist maßgeblich gegenüber allem, was hier auf der Docs-Seite steht — widersprechen sich die beiden, gewinnt die Spec, und die Doku sollte aktualisiert werden.

## Eine Spec hinzufügen oder aktualisieren

Die kanonische Fassung ist die einzige Quelle der Wahrheit. Änderungen fließen so: kanonisches EN → jede Übersetzung neu erzeugen. Eine Übersetzung direkt zu bearbeiten ist nicht der richtige Weg — hebe die Änderung ins Kanonische und erzeuge die Übersetzung neu. Der [Spec-Index](https://github.com/nolte/vale-style/blob/main/spec/README.md) im Repo listet Status und Stand pro Spec.
